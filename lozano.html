-Internet (Creación, concepto, funcionamiento):

Es un conjunto de redes para comunicarse entre sí. La primera red fue en California cuando se estableció conexión entre tres universidades en 1969. Funciona a través de protocolos de comunicación que puede leer e interpretar solo las computadoras.

-Protocolo de comunicación (Concepto,tipos,RFC):

Comunica datos digitales clasificado funcionalmente en la capa de red según el modelo OSI.Su función es de uso bidireccional que transfiere paquetes conmutados a través de diferentes redes físicas enlazadas. Existen diferentes tipos como:

	-TCP/IP					-SSH
        -HTTP					-DNS
        -FTP

Las RFC son un conjunto de documentos que sirven de referencia para la comunidad de Internet, que describen, especifican y asisten en la implementación, estandarización y discusión de la mayoría de las normas.

-Niveles ISO-OSI:

Existen diferentes niveles en el modelo OSI.

         1.- Físico:  Su función es la conexión de equipos.
         2.- Enlace:  Su función es la detección de errores.
         3.- Red:  Su función es el enrutamiento de los mensajes.
         4.- Transporte:  Su función es la integración de los mensajes.
         5.- Sesión:  Su función son los diálogos de control
         6.- Presentación:  Su función es interpretar los datos
         7.- Aplicación:  Su función es mostrar los datos de forma normalizada.

-Protocolo HTTP:

En ingles Hypertext Transfer Protocol, es el protocolo de comunicación que permite las transferencias de información en la World Wide Web, sigue el esquema de petición-respuesta entre un cliente y un servidor. Las principales características de este protocolo son: 

    -Se establece sobre la capa de conexión TCP/IP.
    -Toda comunicación se realiza a partir de caracteres de 8 bits.
    -Respeta el formato original.
    -Permite transferencia de objetos multimedia
    -Cada objeto del protocolo esta identificado a traves de un URL.

Los metodos HTTP indican la accion que se desea realizar para un recurso determinado. Metodos: 

        -GET: Solicita una representación de un recurso específico. 
	-HEAD: Pide una respuesta idéntica a la de una petición GET.
	-POST: Se utiliza para enviar una entidad a un recurso en específico.
	-PUT: Reutiliza para enviar una entidad a un recurso en específico.
	-DELETE: Borra un recurso en especifico.
	-CONNECT: Establece un tunel hacia el servidor.	
	-OPTIONS: Utilizado para describir opciones de comunicación.
	-TRACE: Realiza prueba de bucle de retorno de mensaje.
	-PATCH: Utilizado para aplicar modificaciones parciales a un recurso.

-El proceso de carga de una pagina web:

El primer paso lo hace el navegador que envia una solicitud al servidor para ver la pagina.Durante este proceso hay un tiempo de espera en el que se redirecciona la URL,se busca el dominio, se conecta al servidor, y el servidor responde enviando paquetes.

Una vez ocurrido esto el servidor envia la pagina web, y el navegador la “descarga”.

Y una vez realizados estos pasos la pagina web es cargada, por cada acción que realicemos en la pagina ser repetirá el proceso.

-TCP:

Significa Protoclo de Control de Tranmisiony es un sistema de protocolos que permite la conexión entre ordenadores que no pertenecen a la misma red, ademas de aportar diferentes servicios. Este protocolo garantiza la transferencia de datos confiable. Este protocolo se usa cuando se requiera que no se pierda informacion por el camino.

Este protocolo utiliza diferntes puertos para diferentes servicios, por ejemplo los mas importantes:

        -Puerto 22: SSH,SCP,SFTP. 
	-Puerto 23: Telnet manejo remoto de equipo.
	-Puerto 25: SMTP (Correo).
        -Puerto 37: Time.

-IP:

Es la base fundamental para internet, esta compuesto de datagramas de la fuente del destino. El nivel de transporte parte de el fujo de estos. Se puede dividir el datagrama en segmentos. Es decir permite el desarrollo y transporte de datos, aunque su recepcion no esta asegurada.

-Aplicaciones cliente-servidor vs P2P

La principal diferencia entre Cliente-Servidor y P2P es que en el metodo cliente servidor, toda la información tiene que pasar por el servidor y el servidor se encarga de distribuirlo entre los diferentes clientes conectados, en cambio en P2P no tienen que pasar por ningún servidor, ya que se comunican entre si directamente, se utilizan nodos que se comportan igual entre si.

Ademas en P2P la conexión estará siempre activa. En P2P todos tiene las mismas funciones en cambio en Cliente-Servidor, el servidor tiene funciones que los clientes no tienen.

-¿Qué son los tipos MIME y para qué sirven?

MIME significa Multipurpose Internet Mail Extensions y son las froma estandar de enviar contenido a trabes de la red, estos especifican los tipos de datos. Es decir:

Adjunta a cada archivo un archivo de cabecera donde se indica el tipo y el subtipo del contenido de los datos del archivo. Gracias a esta información tanto el servidor como el navegador pueden manejar y presentar los archivos correctamente.

En sentido general las extensiones de MIME van encaminadas a soportar:

	-Información de encabezados con conjuntos de caracteres distintos.

-Metodologías del desarrollo del software:

Principalmente se refiere a un framework o entorno de trabajo que se usa para estructurar y tener un control del proceso de desarrollo,existen varios pero destacamos los siguientes:

-Modelo cascada: Es dividido en capas y cada etapa representa una unidad de desarrollo con un pequeño descanso en el medio. Por lo tanto, cada siguiente etapa inicia tan pronto como la anterior haya culminado, y esos descansos son usados para confirmaciones del lado del cliente.

-Modelo en espiral: Esta es una metodología para proyectos rápidos con mayor paralelismo a la hora de trabajar y debe ser planificado metódicamente para cada paso del espiral, ya que en este se trabajan múltiples entregas simultáneamente.

-Modelos basados en prototipos: Esta metodología se enfoca en otorgar resultados rápidos, con procesos de desarrollos con herramientas.

-Métodos kanban: Es un sistema basado en funcionamiento de los supermercados que reduce la procrastinacion en todas las áreas, se basa en la facilidad de visualización e interpretación hace que la comunicación entre los funcionarios se vuelva mucho más rápida y efectiva.

-Extreme programming: Su objetivo es el desarrollo y gestión de proyectos con eficacia, flexibilidad y control. Es una metodología basada en la comunicación, la reutilización del código desarrollado y la realimentación.

-Scrum: Se basa en que cada uno es responsable de unas tareas determinadas y de terminarlas en los tiempos acordados. Esto garantiza la entrega de valor del equipo completo, sin necesidad de ayuda o la supervisión minuciosa de otros miembros de la organización.

-Integración continua:

Se basa en una practica de desarrollo de software que consistente en integrar las modificaciones del proyecto automáticamente lo mas a menudo posible para poder detectar fallos cuando antes.

Para ello se pueden usar servidores que generen copias, de forma que el código sea fácil de recuperar a una versión anterior del código, aunque tambien existen paginas web que actúan como servidores.

-Concepto de Pruebas unitarias:Se utiliza para verificar si un método particular en el código de producción está funcionando correctamente, para probar la regresión o para realizar pruebas. Las pruebas ordenadas se utilizan para ejecutar otras pruebas en un orden específico. Existen diferentes herramientas que podemos usar como:

        -Typemock
	-JUnit
	-Qunit

-Los “mock”: Utilizan objetos simulados para reemplazar los objetos reales utilizados por la clase o segmento de código que se va a probar.

-Concepto de Pruebas de integración:

Son aquellas operaciones que se realizan en el campo del desarrollo de software después de la aprobación de las pruebas unitarias y que demuestran que todos los elementos unitarios que componen el software pueden trabajar juntos correctamente al agruparlos.

DB (base de datos) es una colección estructurada de organización lógica y datos relacionados para un propósito específico, que representa entidades y sus relaciones. Un DBMS (sistema de gestión de bases de datos) es un programa informático que le permite crear, modificar y mantener bases de datos.     - DROP, con esta sentencia, podemos eliminar los objetos de la estructura.
    - SELECT, esta sentencia se utiliza para realizar consultas sobre los datos.
    - INSERT, con esta instrucción podemos insertar los valores en una base de datos.
    - UPDATE, sirve para modificar los valores de uno o varios registros.
    - DELETE, se utiliza para eliminar las filas de una tabla.

Programación y organización del código:

Concepto de código spaguetti:

Se refiere a cuando la estructura del programa es compleja y difícil de entender, el código bien estructurado es importante porque ayuda a mantener el programa.

Concepto de abstracción en programación

Implica separar el elemento de su contexto u otros elementos relacionados con él.

Paradigmas de la programación:

El paradigma imperativo es que el programa consta de una serie de instrucciones o un conjunto de oraciones, como si el programador diera un comando específico. Utilizar los principios del razonamiento lógico en el paradigma imperativo para responder preguntas o preguntas formuladas

El paradigma declarado es cuando el programador declara las propiedades del resultado deseado, pero no cómo calcularlo.

Técnicas de programación imperativa:

-Paradigma de programación diseñado para mejorar la claridad, la calidad y el tiempo de desarrollo de los programas informáticos utilizando solo subrutinas y tres estructuras básicas.

-Acoplamiento y cohesion de modulos: Sirve para mejorar la capacidad de mantenimiento de los módulos de software y simplifique el proceso de cambios de software sin tener que ver todos los módulos relacionados.

Programación orientada a objetos.

Principios SOLID: Representa cinco principios básicos de la programación orientada a objetos y el diseño.

        -Principio de responsabilidad única.
        -Principio de abierto/cerrado.
        -Principio de sustitución de Liskov.
        -Principio de segregación de la interfaz.
        -Principio de inversión de la dependencia.

Interfaz: Se considera parte de la interfaz a sus elementos de identificación, de navegación, de contenidos y de acción.

Encapsulacion: Es un mecanismo que consiste en organizar datos y métodos de una estructura, conciliando el modo en que el objeto se implementa

Objetos: Es uno de los elementos esenciales en la programación. Es una unidad dentro de un programa informático que tiene un estado, y un comportamiento.

Herencia: Facilita la creación de objetos a partir de otros ya existentes e implica que una subclase obtiene todo el comportamiento (métodos) y finalmente los atributos

Polimorfismo: Relajación del sistema de tipos que hace que una referencia a una clase (atributo, parámetro o declaración local o elemento de un vector) acepte direcciones de objetos de dicha clase y de sus clases derivadas

Lenguaje compilado vs lenguaje interpretado

En términos generales, la diferencia entre un lenguaje compilado y un lenguaje interpretado es que el primero usa un compilador para traducirlo y ejecutar el programa, mientras que el segundo requiere un intérprete para traducir el código durante la ejecución.

Lenguajes débilmente tipados vs lenguajes fuertemente tipados:

La diferencia entre un lenguaje fuertemente tipado y un lenguaje débilmente tipado es básicamente que un lenguaje fuertemente tipado verifica el tipo de la variable antes de realizar operaciones en la variable, mientras que un lenguaje débilmente tipado no lo hace.

¿Qué diferencia existe entre un tipado estático y uno dinámico?

Es al comprobar la compatibilidad entre tipos. Al verificar los tipos dinámicos en tiempo de ejecución, los tipos estáticos se verificarán después de compilar el programa

¿En qué consiste el type typing?

El estilo de escritura dinámica de datos, donde el método actual y el conjunto de atributos determinan la validez semántica.

¿Qué es la sobrecarga de operadores?

Tiene la capacidad de convertir los operadores del idioma (como +, -, etc.) Cuando se dice que se convierte, esto significa que los operandos en juego no tienen que ser los operandos soportados por el idioma por defecto.

¿Qué es la interpolación de cadenas?

Se basa en la función de formato compuesto y proporciona una gramática más legible y conveniente, que se utiliza para incluir el resultado de la expresión de formato en la cadena de resultados.

Entorno Java:

JVM: Es una máquina virtual de proceso nativo, es decir, ejecutable en una plataforma específica, capaz de interpretar y ejecutar instrucciones expresadas en un código binario especial (el bytecode Java), el cual es generado por el compilador del lenguaje Java.

Concepto de Bytecode:

Es un código intermedio más abstracto que el código máquina. Habitualmente, es tratado como un archivo binario que contiene un programa ejecutable similar a un módulo objeto, que es un archivo binario producido por el compilador cuyo contenido es el código objeto o código máquina .

Concepto Vista:

Una vista es una tabla lógica basada en una tabla u otra vista.

Diagramas de entidad-relación para modelar una base de datos relacional.

El modelo de relación de entidades es una herramienta de modelo de datos que facilita la representación de entidades en la base de datos.

Formas normales

Es un proceso que consiste en designar y aplicar una serie de reglas a las relaciones obtenidas tras el paso del modelo entidad-relación al modelo relacional.

Una tabla está en primera forma si: 

    -Todos los atributos son atómicos. Un atributo es atómico si los elementos del dominio son simples e indivisibles.
    -No debe existir variación en el número de columnas.
    -Los campos no clave deben identificarse por la clave (dependencia funcional).
    -Debe existir una independencia del orden tanto de las filas como de las columnas; es decir, si los datos cambian de orden no deben cambiar sus significados.

Una tabla esta en la segunda forma si está en 1FN y si los atributos que no forman parte de ninguna clave dependen de forma completa de la clave principal.

La tabla se encuentra en 3FN si es 2FN y si no existe ninguna dependencia funcional transitiva en los atributos que no son clave.
