HTTP y Redes

Internet: creación, concepto, funcionamiento

Internet nació en 1969, durante la Guerra Fría, cuando el Departamento de Defensa de EEUU decidió crear un sistema de comunicaciones capaz de resistir un ataque nuclear. El Departamento de Defensa encargó la tarea a la Agencia de Investigación Avanzada de Proyectos de Defensa, que creó un sistema de intercambio de archivos y mensajes llamado Arpanet.

Internet es una red informática descentralizada de alcance global. Es un sistema de redes interconectadas mediante distintos protocolos que ofrece una gran diversidad de servicios y recursos.

Todos los ordenadores conectados en internet tienen que utilizar el mismo protocolo o normas para comunicarse entre ellos, de lo contrario no podrían establecer comunicación.

Protocolo: concepto, tipos, RFC

Un protocolo es un sistema de normas que regulan la comunicación entre dos o más sistemas que se transmiten información a través de diversos medios físicos.

Es decir, son lenguajes o códigos que sirven para garantizar la comunicación entre sistemas informáticos, y tienen una sintaxis y una semántica específicas.

Tipos:
  • Protocolo IP: Protocolo a nivel de red. Su musión es encamminar el datagrama.
  • Protocolo TPC/IP: Conjunto de protocolos de comunicación mediante los cuales se logra la transmisión en redes de ordenadores.
  • Protocolo TPC: Orientado a las comunicaciones, permite la transmisión de datos.
  • Protocolo ARP: Su objetivo es asociar un dispositivo IP a un dispositivo de red.
  • Protocolo HTTP: Diseñado para recuperar información y realizar búsquedas.
  • Protocolo UDP: Destinado a las comunicaciones que se realizan sin conexión.
  • Protocolo ICMP: Cuando ocurre algo inesperado, este protocolo informa del suceso.
  • Protocolo SMTP: Conjunto de reglas que rigen el formato y la transferencia de datos en un envío de Correo Electrónico.
  • Protocolo SSH: Desarrollado para mejorar la seguridad, lo que hace es eliminar el envío de contraseñas no cifradas, codificando toda la información transferida.
RFC:

Request for Comments, RFC, son una serie de documentos en los que se describen y definen protocolos, conceptos, métodos y programas de Internet.

Niveles ISO-OSI

Protocolo HTTP: concepto, utilidad, caractrísticas

HTTP (Hypertext Transfer Protocol) es un protocolo de acceso para las páginas web a través de Internet.

Las principales caractrísticas del protocolo HTTP son:
  • Toda la comunicación entre los clientes y servidores se realiza a partir de caracteres de 8 bits. De esta forma, se puede transmitir cualquier tipo de documento: texto, binario, etc., respetando su formato original.
  • Permite la transferencia de objetos multimedia. El contenido de cada objeto intercambiado está identificado por su clasificación MIME.
  • Existen tres verbos básicos (hay más, pero por lo general no se utilizan) que un cliente puede utilizar para dialogar con el servidor: GET, para recoger un objeto, POST, para enviar información al servidor y HEAD, para solicitar las características de un objeto (por ejemplo, la fecha de modificación de un documento HTML).
  • Cada operación HTTP implica una conexión con el servidor, que es liberada al término de la misma. Es decir, en una operación se puede recoger un único objeto.
  • No mantiene estado. Cada petición de un cliente a un servidor no es influida por las transacciones anteriores. El servidor trata cada petición como una operación totalmente independiente del resto.
  • Cada objeto al que se aplican los verbos del protocolo está identificado a través de la información de situación del final de la URL.


Estructura de un paquete HTTP de petición y de respuesta

Las peticiones y respuestas HTTP, comparten una estructura similar, compuesta de:

  • Una línea de inicio ('start-line' en inglés) describiendo la petición a ser implementada, o su estado, sea de éxito o fracaso. Esta línea de comienzo, es siempre una única línea.
  • Un grupo opcional de cabeceras HTTP, indicando la petición o describiendo el cuerpo ('body' en inglés) que se incluye en el mensaje.
  • Una línea vacía ('empty-line' en inglés) indicando toda la meta-información ha sido enviada.
  • Un campo de cuerpo de mensaje opcional ('body' en inglés) que lleva los datos asociados con la petición (como contenido de un formulario HTML), o los archivos o documentos asociados a una respuesta (como una página HTML, o un archivo de audio, vídeo ... ) . La presencia del cuerpo y su tamaño es indicada en la línea de inicio y las cabeceras HTTP

Métodos HTTP y su utilidad

HTTP define un conjunto de métodos de petición para indicar la acción que se desea realizar para un recurso determinado.

  • GET: Solicita una representación de un recurso específico. Las peticiones que usan el método GET sólo deben recuperar datos.
  • HEAD: Pide una respuesta idéntica a la de una petición GET, pero sin el cuerpo de la respuesta.
  • POST: Se utiliza para enviar una entidad a un recurso en específico, causando a menudo un cambio en el estado o efectos secundarios en el servidor.
  • PUT: Reemplaza todas las representaciones actuales del recurso de destino con la carga útil de la petición.
  • DELETE: Borra un recurso en específico.
  • CONNECT: Establece un túnel hacia el servidor identificado por el recurso.
  • OPTIONS: Es utilizado para describir las opciones de comunicación para el recurso de destino.
  • TRACE: Realiza una prueba de bucle de retorno de mensaje a lo largo de la ruta al recurso de destino.
  • PATH: Utilizado para aplicar modificaciones parciales a un recurso.

Proceso de carga de una página web

El primer paso es introducir una dirección en la barra de búsqueda de un navegador (la URL). Para que los datos lleguen desde el servidor al navegador se utiliza el protocolo de transferencia de hipertexto o Hypertext Transfer Protocol (HTTP) y su variante encriptada HTTPS, protocolo seguro de transferencia de hipertexto o Hypertext Transfer Protocol Secure. Para que un navegador pueda acceder a los contenidos de una web, la URL debe convertirse en una dirección IP. De esto se encargan los llamados servidores DNS, responsables de la gestión del sistema de nombres de dominio o DNS (Domain Name Server). Cuando introducimos una dirección en la barra de búsqueda de un navegador, este dirige su URL a un router, el cual se encarga de buscar la dirección IP correspondiente para esta página web. La información necesaria se la proporciona un servidor DNS, un servidor web especial responsable de la resolución de nombres.

Una vez el router ha encontrado la dirección IP de la página solicitada, pide en el servidor web correspondiente los datos necesarios para mostrar la página en el navegador. Esta consulta tiene lugar mediante HTTP en la forma de un paquete de datos que contiene toda la información que el servidor web necesita para entregar los datos de la página web. Además de la dirección IP de la página web solicitada, el router comunica su propia dirección IP como emisor y proporciona información sobre el sistema operativo, el navegador y el tipo de dispositivo que ha de mostrar la página web. El servidor web evalúa esta información y emite un código de estado HTTP. Si la solicitud tiene éxito, el servidor envía un paquete de datos al navegador con toda la información necesaria para la visualización de la página web. Si, por el contrario, el servidor no encuentra la página web en la dirección solicitada, o bien emite un código de error 404 (página web no encontrada) o redirige a la nueva dirección si la conoce.

TCP: concepto, utilidad, puertos bien conocidos, estructura de los segmentos TCP, sockets

El Protocolo de Control de Transmisión, TCP, se utiliza para garantizar que los datos se entregan al destino sin errores y en el mismo orden en que son transmitidos. Además, proporciona un mecanismo para distinguir aplicaciones dentro de la misma máquina, a través de los puertos.

Puertos bien conocidos: Son puertos específicos utilizados por el sistema o por procesos con privilegios. Los asigna la Internet Assigned Numbers Authority (IANA). Van del 0 al 1023, por ejemplo: SSH (22), HTTP (80), FTP (21).

Los segmentos TCP son paquetes de bits que constituyen las unidades de datos del protocolo TCP.

TCP se ocupa de convertir el flujo de datos saliente de una aplicación de forma que se pueda entregar como fragmentos. La aplicación traslada los datos a TCP y éste sitúa los datos en un buffer de envío. TCP toma un trozo de esos datos y le añade una cabecera, creando de esta forma un segmento. Este segmento es trasladado a IP para que lo entregue como un único datagrama.

Los sockets constituyen el mecanismo para la entrega de paqeutes de datos provenientes de la tarjeta de rede a los procesos apropiados. Está definido por un par de direcciones IP (local y remota), un protocolo de transporte y un par de direcciones de puerto (local y remoto).

IP: concepto, utilidad, direcciones IP, enrutamiento, estructura de los datagramas IP)

La dirección IP es un conjunto de números que identifica, de manera lógica y jerárquica, a una interfaz en la red (elemento de comunicación/conexión) de un dispositivo (computadora, laptop, teléfono inteligente) que utilice el protocolo (Internet Protocol) o, que corresponde al nivel de red del modelo TCP/IP.

Los dispositivos se conectan entre sí mediante sus respectivas direcciones IP, pero para los usuarios es más sencillo recordar un nombre que un conjunto de números, por lo que se utiliza un nombre de dominio.

Una tabla de enrutamiento es un conjunto de reglas que sirven para determinar qué camino deben seguir los paquetes de datos. Todo esto a través de toda red que trabaje con el protocolo IP.

Los componentes de una tabla de enrutamiento son:

  • Red de destino: corresponde a la red de destino donde deberá ir el paquete de datos.
  • Máscara de subred: es la que se utiliza para definir la máscara de subred de la red a la que debemos ir.
  • Siguiente salto: dirección de IP de la interfaz de red por donde viajará el paquete de datos, para seguir con su camino hasta el final.
  • Interfaz de salida: es la interfaz de red por donde deben salir los paquetes, para posteriormente llegar finalmente al destino.
  • Métricas

Un datagrama es un paquete de datos que constituye el mínimo bloque de información en una red de conmutación por datagramas, la cual es uno de los dos tipos de protocolo de comunicación por conmutación de paquetes usados para encaminar por rutas diversas dichas unidades de información entre nodos de una red, por lo que se dice que no está orientado a conexión.

Los datagramas se componen de:

  • una cabecera con información de control
  • los propios datos que se desean transmitir

URL: concepto, estructura, parámetros

Un URL, o localizador de recursos uniforme, es un identificador de recursos uniforme (Uniform Resource Identifier, URI) cuyos recursos referidos pueden cambiar, esto es, la dirección puede apuntar a recursos variables en el tiempo.

Están formados por una secuencia de caracteres de acuerdo con un formato modélico y estándar que designa recursos en una red como, por ejemplo, Internet. El formato general de una URL es: 

:esquema://máquina/directorio/archivo

Un parámetro de URL es una propiedad que agrega un creador a la configuración de un cuadro de mando y que se puede usar para desencadenar acciones. Un parámetro de URL es un par nombre-valor. En cada parámetro, el nombre y el valor se separan por el signo igual (=). Durante el tiempo de ejecución, puede agregar un parámetro al final de la URL del cuadro de mando con un signo de interrogación (?) seguido del parámetro.

Aplicaciones cliente-servidor vs P2P

Cliente-Servidor: es un modelo de aplicación distribuida en el que las tareas se reparten entre los proveedores de recursos o servicios, llamados servidores, y los demandantes, llamados clientes. Un cliente realiza peticiones a otro programa, el servidor, quien le da respuesta.

P2P: es una red de computadoras en la que todos o algunos aspectos funcionan sin clientes ni servidores fijos, sino una serie de nodos que se comportan como iguales entre sí. Es decir, actúan simultáneamente como clientes y servidores respecto a los demás nodos de la red.

Cookies: definición y utilidad. ¿Cómo establecer una sesión utilizándolas?

Una cookie es un archivo de pequeño tamaño enviado por un sitio web y almacenado en el navegador del usuario, de manera que el sitio web puede consultar la actividad previa del navegador. Así, es posible identificar al usuario que visita un sitio web y llevar un registro de su actividad en el mismo.

Una sesión es un mecanismo de programación de las tecnologías de web scripting que permite conservar información sobre un usuario al pasar de una página a otra. A diferencia de una cookie, los datos asociados a una sesión se almacenan en el servidor y nunca en el cliente. En la mayoría de las tecnologías de web scripting, las sesiones se implementan mediante una cookie que almacena un valor que identifica al usuario en el servidor web cada vez que pasa de una página web a otra. En el servidor web están almacenados todos los datos de la sesión y se accede a ellos cada vez que se pasa de página gracias al identificador almacenado en la cookie.

¿Qué son los tipos MIME y para qué sirven?

Son la manera estándar de mandar contenido a través de la red. Los tipos MIME especifican tipos de datos, como por ejemplo texto, imagen, audio, etc. que los archivos contienen.



PROYECTOS WEB

Requerimientos, especificaciones de una aplicación

A la hora de realizar un una aplicación web es muy importante realizar una serie de documentos en los que se describan los requerimientos y especificaciones de lo que se desea conseguir, tanto a nivel funcional, como en lo referido al diseño. De esta forma, una vez se haya finalizado el proyecto, se podrá comprobar que se cumplen todos los objetivos marcados inicialmente, lo cual significaría haber conseguido un software de calidad.

Metodologías del desarrollo del software

  • Modelo en cascada: Es el enfoque metodológico que ordena rigurosamente las etapas del proceso para el desarrollo de software, de tal forma que el inicio de cada etapa debe esperar a la finalización de la etapa anterior.

    Sus fases son:

    • Análisis de requisitos del software
    • Diseño del sistema
    • Diseño del programa
    • Codificación
    • Pruebas y error
    • Validación y Verificación del producto de software
    • Mantenimiento

  • Modelo en espiral: Es una combinación entre el modelo lineal o de cascada y el modelo iterativo. consiste en seguir ciclos crecientes de cuatro fases cada uno, que se van realizando siguiendo una forma de espiral. En cada ciclo se pasa por dichas fases bien definidas, como en el modelo de cascada, pero con capacidad de evolucionar su complejidad con cada ciclo.

    Sus fases son:

    • Planificación
    • Análisis de Riesgo
    • Implementación
    • Evaluación

  • Modelos basados en prototipos: Es un estilo de programación orientada a objetos en el cual los objetos no son creados mediante la instanciación de clases sino mediante la clonación de otros objetos o mediante la escritura de código por parte del programador. De esta forma los objetos ya existentes pueden servir de prototipos para los que el programador necesite crear.

    Sus etapas son:

    • Recolección y refinamiento de requisitos
    • Modelado, diseño rápido
    • Construcción del Prototipo
    • Desarrollo, evaluación del prototipo por el cliente
    • Refinamiento del prototipo
    • Producto de Ingeniería

  • Metodologías ágiles:
    • Métodos Kanban: Se trata de un sistema de gestión y control de inventario y flujo de piezas que utiliza pequeñas hojas de papel autoadhesivo, conocidas como post-its. El sistema Kanban se basa, principalmente, en referencias visuales. Los papeles coloridos son colocados en productos, en los espacios de la empresa, murales e incluso en las computadoras que tienen un programa electrónico del Kanban.
    • Extreme programming: Su objetivo es el desarrollo y gestión de proyectos con eficacia, flexibilidad y control.
    • Scrum: Es un proceso en el que se aplican de manera regular un conjunto de buenas prácticas para trabajar colaborativamente, en equipo, y obtener el mejor resultado posible de un proyecto.

      Sprint es un intervalo prefijado de tiempo (no inferior a una semana ni superior a un mes) durante el cual se crea un incremento de producto "Hecho o Terminado" utilizable, potencialmente entregable.

      El diagrama burndown es una representación gráfica del trabajo por hacer en un proyecto en el tiempo.

      Los tres principales artefactos o herramientas Scrum son: el Product Backlog, Sprint Backlog y el Incremento.

Integración continua: concepto, servidores

Es una práctica de ingeniería de software que consiste en hacer integraciones automáticas de un proyecto lo más a menudo posible para así poder detectar fallos cuanto antes.

  • Pruebas unitarias: Es una forma de comprobar el correcto funcionamiento de una unidad de código. Para realizar estas pruebas se utilizan mocks, que son objetos falsos que simulan el comportamiento de objetos reales.
  • Pruebas de integración: Se realizan tras las pruebas unitarias, para comprobar que todos los elementos unitarios que componen el software funcionan de manera global. Comprueba la comunicación entre los componentes de la aplicación.
  • TDD: El Desarrollo dirigido por Tests es una metodología de desarrollo cuyo objetivo es crear primero las pruebas y luego escribir el software.

Documentación de programas: concepto, tipos, herramientas

La documentación sobre una aplicación tiene como objeto asegurar que tanto el desarrollador como el cliente tienen la misma idea sobre las funcionalidades del sistema. Es muy importante, ya que si no se especifican las características y requerimientos, el software puede no ser aceptado por el cliente.

Los pasos que se siguen para documentar un programa son los siguientes:

  • Análisis de factibilidad
  • Análisis de requerimientos
  • Diseño del sistema
  • Implementación
  • Validación y pruebas
  • Explotación
  • Mantenimiento

Algunas herramientas de documentación técnica son: Huddle, Read the Docs, Javadoc, Markdown o Swagger.

Despliegue

  • Despliegue en producción y en test: El despliegue en producción requiere la instalación y configuración de todos los elementos que vayan a ser necesarios para que los componentes funcionen, tanto software como hardware. El despliegue en test es útil para corregir y prevenir problemas.
  • Despliegue automatizado: En este se incluyen mecanismos y herramientas necesarias para realizar el despliegue de manera automática, sin que intervenga una persona.
  • Despliegue en la nube: Se realiza en una nube, ya sea pública, privada o híbrida, de forma que los recursos no se encuentran en local, sino en un servidor remoto.

Contenedores

Un contenedor es un paquete de software estándar que agrupa el código de una aplicación con las bibliotecas y los archivos de configuración asociados, junto con las dependencias necesarias para que la aplicación se ejecute. Esto permite a los desarrolladores y profesionales de TI implementar aplicaciones sin problemas en todos los entornos.

Gestores de versiones: concepto, herramientas y características

El control de versiones hace referencia a la gestión de los cambios que se realizan en un proyecto.

Un sistema de control de versiones debe proporcionar:

  • Mecanismo de almacenamiento de los elementos que deba gestionar (ej. archivos de texto)
  • Posibilidad de realizar cambios sobre los elementos almacenados
  • Registro histórico de las acciones realizadas con cada elemento o conjunto de elementos (normalmente pudiendo volver o extraer un estado anterior del producto).

Algunas herramientas de gestión de versiones son Git, CVS, Apache Subversion (SVN), Mercurial o Monotone.

Repositorio de código: tipos, herramientas

Un repositorio de código es un lugar donde el código de una aplicación, de un programa cualquiera está almacenado y desde donde se puede distribuir. Estos repositorios están almacenados en servidores, no en ordenadores locales o en un disco duro que tengamos.

Algunas herramientas son GitHub, GitLab, SourceForge, GitKraken o Apache Allura.

Evaluación de la calidad del software: concepto, herramientas disponibles

Cuando se cumplen los requerimientos y estándares de desarrollo establecidos en la primera fase del planteamiento de un proyecto con el software ya terminado, se habla de un software de calidad.

Cuando se examina un elemento sus características mensurables se pueden encontrar dos tipos de calidad:

  • Calidad de diseño: se refiere a las características que los diseñadores especifican para un elemento.
  • Calidad de concordancia: grado en el que las especificaciones de diseño se aplican durante la fabricación.

Algunas herramientas para la evaluación de la calidad son PMD, Check Style, SONAR, Google CodePro Analytix o Simian.

DISEÑO DE APLICACIONES WEB

API de Servidor

API es el acrónimo utilizado para Application Programming Interface, o en español, Interfaz de Programación de Aplicaciones. Se trata de una interfaz que permite la comunicación entre dos sistemas o plataformas diferentes, lo que permite agregar diversas funciones a sitios web y aplicaciones.

Diagrama Entidad-Relación

También conocido como modelo entidad relación o ERD, es un tipo de diagrama de flujo que ilustra cómo las "entidades" se relacionan entre sí dentro de un sistema. Los diagramas ER se usan a menudo para diseñar o depurar bases de datos relacionales.

Normalización: primera, segunda y tercera forma normales

La normalización de bases de datos es un proceso que consiste en designar y aplicar una serie de reglas a las relaciones obtenidas tras el paso del modelo entidad-relación al modelo relacional. Con objeto de minimizar la redundancia de datos, facilitando su gestión posterior.

Primera Forma Normal
Las reglas de la Primera Forma Normal son las siguientes:
  • Todos los atributos son atómicos.
  • Cada tabla debe contener una clave primaria única.
  • La clave primaria no contiene atributos nulos.
  • No debe existir variación en el número de columnas.
  • Debe existir independencia del orden de filas y columnas.

Segunda Forma Normal
Una tabla está en 2FN si además de estar en 1FN cumple que los atributos no clave depende de TODA la clave principal.

Tercera Forma Normal
Una tabla está en 3FN si además de estar en 2FN no existe ninguna dependencia transitiva entre los atributos que no son clave.

Restricciones de integridad

Una restricción de integridad representa una propiedad del mundo real del cual la base de datos es una representación.

Las restricciones de integridad se especifican en el esquema de la base de datos y se deben cumplir en cualquier extensión del mismo

En el modelo relacional se contemplan cuatro tipos de restricciones:

  • Restricción de unicidad
  • Restricción de valor no nulo
  • Restricción de clave primaria
  • Restricción de integridad referencial

El DDL, Data Manipulation Language, es un lenguaje proporcionado por el sistema de gestión de base de datos que permite a los usuarios llevar a cabo las tareas de consulta o manipulación de los datos, organizados por el modelo de datos adecuado.

El DDL, Data Definition Language, es el lenguaje encargado de la modificación de la estructura de los objetos de la base de datos. Incluye órdenes para modificar, borrar o definir las tablas en las que se almacenan los datos de la base de datos. Existen cuatro operaciones básicas: CREATE, ALTER, DROP y TRUNCATE.

Paradigmas de la programación

¿En qué consiste paradigma declarativo frente al paradigma imperativo?

¿Qué es la programación funcional y en qué se diferencia respecto de la programación imperativa?

Técnicas de programación imperativa

Programación estructurada
Es un paradigma de programación orientado a mejorar la claridad, calidad y tiempo de desarrollo de un programa de computadora recurriendo únicamente a subrutinas y tres estructuras básicas: secuencia, selección (if y switch) e iteración (bucles for y while).

Programación modular
Es un paradigma de programación que consiste en dividir un programa en módulos o subprogramas con el fin de hacerlo más legible y manejable. Un 'módulo' es cada una de las partes de un programa que resuelve uno de los subproblemas en que se divide el problema complejo original. Cada uno de estos módulos tiene una tarea bien definida y algunos necesitan de otros para poder operar.

Programación orientada a objetos
  • Principios SOLID: Se trata de 5 principios cuyo objetivo es crear un software eficaz, con un código limpio y flexible, que sea reutilizable y mantenible.
  • Interfaces: es un medio común para que los objetos no relacionados se comuniquen entre sí. Estas son definiciones de métodos y valores sobre los cuales los objetos están de acuerdo para cooperar
  • Clases
    • Clases abstractas: Las clases abstractas son aquellas que por sí mismas no se pueden identificar con algo 'concreto' (no existen como tal en el mundo real), pero sí poseen determinadas características que son comunes en otras clases que pueden ser creadas a partir de ellas.
    • Propiedades y Métodos: Visibilidad de los métodos y de las propiedades - Las propiedades de un objeto son las características que tiene, mientras que los métodos son algoritmos asociados, que pueden producir cambios en las propiedades de dicho objeto. Pueden ser public, private o protected.
    • Implementación de interfaces: Para declarar una clase que implemente una interfaz es necesario utilizar la palabra reservada implements en la cabecera de declaración de la clase
  • Concepto de encapsulación: Se refiere ocultamiento de los datos miembro de un objeto de manera que solo se pueda cambiar mediante las operaciones definidas para ese objeto.
  • Objetos: Un objeto en POO representa alguna entidad de la vida real, es decir, alguno de los objetos únicos que pertenecen al problema con el que nos estamos enfrentando, y con el que podemos interactuar. Cada objeto, de igual modo que la entidad de la vida real a la que representa, tiene un estado (es decir, unos atributos con unos valores concretos) y un comportamiento (es decir, tiene funcionalidades o sabe hacer unas acciones concretas).
  • Herencia: Concepto de Sobreescribir un método - La herencia (a la que habitualmente se denomina subclase) proviene del hecho de que la subclase (la nueva clase creada) contiene las atributos y métodos de la clase primaria. La sobreescritura de métodos nos permite redifinir un método que heredamos para que este funcione de acuerdo a nuestras necesidades y no a lo definido en la superclase.
  • Concepto de Polimorfismo: se refiere a la propiedad por la que es posible enviar mensajes sintácticamente iguales a objetos de tipos distintos
    • polimorfismo basado en implementación de interfaces
    • polimorfismo basado en la herencia

Explica en qué consisten las Interfaces fluidas
Es una construcción orientada a objetos que define un comportamiento capaz de retransmitir el contexto de la instrucción de una llamada subsecuente. Definido a través del valor de retorno de un método llamado autoreferencial, donde el nuevo contexto es equivalente al contexto anterior terminado por medio del retorno de un contexto vacío (void context).

Qué es la Inyección de dependencias o inversión del control y qué ventajas aporta su uso en la programación web
La inyección de dependencias es un patrón de diseño de software usado en la Programación Orientada a Objetos, que trata de solucionar las necesidades de creación de los objetos de una manera práctica, útil, escalable y con una alta versatilidad del código. En la mayoría de los frameworks actuales se aplica la Inyección de dependencias como parte de las herramientas y modelos que facilitan al programador.

Qué es la programación orientada a aspectos y qué ventajas aporta su uso en la programación web
Es un paradigma de programación que basa su filosofía en tratar las obligaciones transversales de los programas como módulos separados (aspectos) para lograr una correcta separación de responsabilidades. Una obligación transversal es aquella que se repite en varias partes de un programa independientemente de si las secciones en las que aparece tienen relación directa.

Ventajas:

  • Óptima para modularizar programas
  • Código fuente más ordenado
  • Puede mezclarse con cualquier otro paradigma de programación

¿Para qué sirven las excepciones y su tratamiento en un programa?
Una excepción en términos de lenguaje de programación es la indicación de un problema que ocurre durante la ejecución de un programa.

El manejo de excepciones ayuda al programador a trasladar el código para manejo de errores de la línea principal de ejecución, además se puede elegir entre manejar todas las excepciones, las de cierto tipo o de las de grupos relacionados, esto hace que la probabilidad de pasar por alto los errores se reduzca y a la vez hace los programas más robustos.

Genericidad del código:
Explica la siguiente regla: Cuanto más genéricos son los programas, más reusable es el código pero también es más complejo y difícil de mantener

La genericidad es un mecanismo que consiste en estructurar el código de forma que se programen funciones o módulos que puedan servir para diferentes partes de una aplicación.

De esta forma, si se quiere cambiar un elemento común en varias secciones, sólo habría que hacer el cambio en el elemento principal, que estaría insertado en dichas secciones.

Lenguajes de programación

Lenguajes compilados vs interpretados
El lenguaje compilado requiere un paso adicional antes de ser ejecutado, la compilación, que convierte el código que escribes a lenguaje de máquina. Un lenguaje interpretado, por otro lado, es convertido a lenguaje de máquina a medida que es ejecutado.
  • Ejemplos de lenguajes compilados: C, C++, Java, Go y Rust.
  • Ejemplos de lenguajes interpretados: Ruby, Python y JavaScript

Diferencias entre un Compilador, un interprete y un transpilador
  • Un compilador es un programa informático que traduce todo el código fuente de un proyecto de software a código máquina antes de ejecutarlo.
  • Un intérprete es un programa informático que procesa el código fuente de un proyecto de software durante su tiempo de ejecución, es decir, mientras el software se está ejecutando, y actúa como una interfaz entre ese proyecto y el procesador.
  • Un transpilador es un tipo especial de compilador que traduce de un lenguaje fuente a otro fuente también de un nivel de abstracción parecido. Se diferencia de los compiladores tradicionales en que estos reciben como entrada ficheros conteniendo código fuente y generan código máquina del más bajo nivel.

Lenguajes débilmente tipados vs lenguajes fuertemente tipados

Un lenguaje fuertemente tipado tiene reglas de tipeo más estrictas en el momento de la compilación, lo que implica que es más probable que ocurran errores y excepciones durante la compilación. La mayoría de estas reglas afectan la asignación de variables, los valores de retorno y la llamada a funciones. Ventajas:

  • Menor número de errores
  • Mayor seguridad a la hora de hacer operaciones y comparaciones
Desventajas:
  • Mayor complejidad y extensión de código

Por otro lado, un lenguaje débilmente escrito tiene reglas de escritura más flexibles y puede producir resultados impredecibles o puede realizar una conversión de tipo implícita en tiempo de ejecución. Ventajas:

  • Permite cambiar el tipo de variable en cualquier momento, sin tener que realizar variaciones en el código
  • Menos código
Desventajas:
  • A la hora de hacer operaciones puede haber problemas
  • Al comparar números puede haber errores ya que compara cadenas
  • Mayor vulnerabilidad

¿Qué diferencia existe entre un tipado estático y uno dinámico?

Un lenguaje es de tipado estático cuando es necesario definir los tipos en tiempo de compilación para que el programa funcione. De igual forma, si se llama a un método o función que no existe, el compilador mostrará error y no ejecutará el programa.

Un lenguaje de tipado dinámico permite definir los tipos de las variables en tiempo de ejecución. El ejemplo que se suele dar es que una variable puede contener valores de distintos tipos durante la ejecución del programa.

¿Qué es el timetomarket de un lenguaje?

Time to market es un término de marketing que se refiere al momento en el que un producto está listo para salir al mercado.

¿Qué es la sobrecarga de operadores?

Se trata de reutilizar el mismo operador con un número de usos diferentes. La sobrecarga de operadores solo se puede utilizar con clases, no se pueden redefinir los operadores para los tipos simples predefinidos.

¿Qué es la interpolación de cadenas?

La interpolación de cadenas es el proceso de evaluar una cadena literal que contiene uno o más marcadores de posición, lo que produce un resultado en el que los marcadores de posición se reemplazan con sus valores correspondientes.

  • Ejemplo en Java: 
    int apples = 4; 
int bananas = 3; 
System.out.println(String.format("I have %s apples and %s bananas", apples, bananas)); 
System.out.println(String.format("I have %s fruit", apples + bananas));
  • Ejemplo en JavaScript: 
    var apples = 4; 
var bananas = 3; 
console.log(`I have ${apples} apples`); 
console.log(`I have ${apples + bananas} fruit`);

La diferencia entre ambos radica en que JDK se utiliza para el desarrollo de un programa, mientras que JRE permite su ejecución.

Otros módulos de spring

Spring Framework esta compuesto de diversos módulos y cada uno de estos módulos aporta una funcionalidad al framework. Los más importantes son los que pertenecen a lo que se llama el Spring Core Container:

  • spring-core: encargado de implementar el código del IoC de Spring permitiendo inyectar beans.
  • spring-beans: encargado de añadir las típicas Factorías como BeanFactory, encargada de instanciar los distintos beans registrados en el framework.
  • spring-context: encargado de añadir un contexto al framework y se encarga por ejemplo de simplificar la carga de los distintos recursos.
  • spring-context-support: encargado de añadir las clases que gestionan servicios complementarios.
  • spring-expression: encargado de añadir a Spring su Expresion Language para poder usarlo en los ficheros xml o anotaciones.

Contenedores de servlets Java

  • Jetty: servidor HTTP 100% basado en Java y un contenedor de Servlets escrito en Java.
  • Tomcat: funciona como un contenedor de servlets desarrollado bajo el proyecto Jakarta en la Apache Software Foundation. Tomcat implementa las especificaciones de los servlets y de JavaServer Pages (JSP) de Oracle Corporation.
  • Undertow: servidor web escrito en Java que ofrece APIS blocking y non-blocking sobre NIO. Tiene una arquitectura de composición que permite construir un servidor web combinando handlers.

Concepto de servlet

Un servlet es una clase en el lenguaje de programación Java, utilizada para ampliar las capacidades de un servidor. Aunque los servlets pueden responder a cualquier tipo de solicitudes, estos son utilizados comúnmente para extender las aplicaciones alojadas por servidores web, de tal manera que pueden ser vistos como applets de Java que se ejecutan en servidores en vez de navegadores web.

  • Objetos Request: El navegador envía información al servidor cada vez que navegamos a una pagina, como el lenguaje utilizado por el navegador, el tipo de navegador que estas usando, etc.
  • Objetos Response: Datos que se envíane en la cabecera de respuesta ante la petición realizada.

Métodos service e init
El método init() es el que inicializa el servlet por defecto. Una vez hecho, cada petición de usuario lanza un hilo que llama al método service(), el cual obtiene el tipo de petición que se ha realizado (GET, POST, PUT, DELETE).

El descriptor de despliegue web.xml
Un descriptor de despliegue (DD) es un componente de aplicaciones J2EE que describe cómo se debe desplegar (o implantar) una aplicación web. En aplicaciones J2EE, XML se usa para la sintaxis del fichero descriptor de despliegue. Debe ser llamado web.xml, y debe ser colocado en un subdirectorio llamado WEB-INF.

Sesiones
HTTP es un protocolo sin sesión, por lo que no se puede decir desde un acceso al servidor a otro si se trata de la misma persona que está accediendo repetidamente al sitio, o si se trata de una persona diferente.

Una sesión es una o más solicitudes de páginas por parte de un cliente a un sitio web durante un periodo definido de tiempo.

Para saber si la sesión es nueva o una ya iniciada, se utilizan cookies, de forma que la clase Session del API servlet usa la clase Cookie.

Sistemas para gestionar logs en Java

  • Log4j: Biblioteca open source desarrollada en Java por la Apache Software Foundation que permite a los desarrolladores de software escribir mensajes de registro, cuyo propósito es dejar constancia de una determinada transacción en tiempo de ejecución. Log4j permite filtrar los mensajes en función de su importancia.
  • Log4j2: A las características anteriores, añade, entre otras novedades, soporte para JMX, el conjunto de herramientas de Java que permite, entre otras cosas, administrar las aplicaciones Java
  • LogBack: Sistema de log similar al conocido Log4j, que pretende mejorarlo y convertirse en su sucesor. Logback ha sido desarrollado por la misma que gente que desarrolló log4j, tratando de corregir los problemas de este último.

Concepto de campos autonuméricos. Claves. Claves ajenas

Los campos autonuméricos son los utilizados para identificar las claves de una tabla, de forma que Cada fila tendrá un valor numérico que no se repetirá nunca e irá incrementándose en 1 con cada inserción.

Una clave ajena es una limitación referencial entre dos tablas.

Concepto de vista

Es el conjunto de resultados de una consulta almacenada en los datos. es una consulta que se presenta como una tabla (virtual) a partir de un conjunto de tablas en una base de datos relacional.

Diagramas de entidad-relación para modelar una base de datos relacional

Los diagramas ER son un lenguaje gráfico para describir conceptos.

Para elaborarlos se utiliza una serie de elementos:

  • Entidades: son el fundamento del modelo entidad relación. Se trata de cualquier cosa o parte del mundo que es distinguible del resto. Se representan por medio de un rectángulo.
  • Atributos: Se representan mediante un círculo o elipse etiquetado mediante un nombre en su interior, aunque no suelen aparecer representados en los diagramas E/R.
  • Relación: Describe cierta dependencia entre entidades o permite la asociación de las mismas. Se representa con una flecha entre las entidades.

Lenguaje de consulta SQL

El lenguaje de manipulación de datos (DML) consta de 4 comandos principales:

  • SELECT: Permite visualizar la información de la base de datos.
  • INSERT: Permite introducir datos a la base de datos.
  • UPDATE: Permite editar información existente.
  • DELETE: Permite eliminar datos.

Resultset

Un ResultSet contiene todas las filas que satisfacen las condiciones de una sentencia SQL y proporciona el acceso a los datos de estas filas mediante un conjunto de métodos get que permiten el acceso a las diferentes columnas de la filas. El método ResultSet.next se usa para moverse a la siguiente fila del result set, convirtiendo a ésta en la fila actual. Simplificando, un Result Set es una tabla, pero no hay garantía de que los datos lleguen en orden, por lo que se puede usar la sentencia ORDER BY para asignar un determinado orden establecido en las filas del ResultSet.

Transacciones

Una transacción es una interacción con una estructura de datos compleja, compuesta por varios procesos que se han de aplicar uno después del otro. La transacción debe realizarse de una sola vez y sin que la estructura a medio manipular pueda ser alcanzada por el resto del sistema hasta que se hayan finalizado todos sus procesos.

Prepared statements y la inyección de código

Son plantillas para consultas a sistemas de bases de datos en lenguaje SQL cuyos parámetros están desprovistos de valores. Para reemplazar dichos valores, estas plantillas trabajan con variables o marcadores de posición, que no son sustituidos por los valores reales hasta estar dentro del sistema.

Un defecto de inyección de código ocurre cuando es posible enviar datos inesperados a un intérprete. Una inyección de código puede resultar en la pérdida o corrupción de datos, falta de responsabilidad en acciones o denegación de acceso. Una inyección puede incluso tomar control total de un nodo. Una inyección de código SQL se aprovecha de la sintaxis en este lenguaje para introducir comandos de manera ilícita que permitan leer o modificar la base de datos, comprometiendo el contenido de la consulta original. Con las Prepared Statments se evita este riesgo.

CONTROL DE VERSIONES

Se llama control de versiones a la gestión de los diversos cambios que se realizan sobre los elementos de algún producto o una configuración del mismo. Una versión, revisión o edición de un producto, es el estado en el que se encuentra el mismo en un momento dado de su desarrollo o modificación.

Ventajas que aportan los controladores de versiones

  • Trazabilidad: seguimiento de un producto desde que se inicia el proceso de su fabricación hasta que llega a manos del consumidor final.
  • Historial de cambios: Característica que permite controlar la evolución de una aplicación y los cambios que se han ido realizando, pudiendo ver en todo momento quién lo ha hecho, comparar versiones o cuándo se ha introducido un error.
  • Reversión: Característica que permite volver al estado anterior de una aplicación cuando hay un error.
  • Trabajo en grupo
  • Trabajo en ramas: Es una forma de separar nuevo trabajo del trabajo ya existente en el repositorio.

Herramientas

  • CVS: Concurrent Versions System es una aplicación informática que implementa un sistema de control de versiones: mantiene el registro de todo el trabajo y los cambios en los ficheros.
  • Apache Subversion: herramienta de control de versiones open source basada en un repositorio cuyo funcionamiento se asemeja enormemente al de un sistema de ficheros.
  • GIT: software de control de versiones, cuyo propósito es llevar un registro de los cambios en archivos de computadora y coordinar el trabajo que varias personas realizan sobre archivos compartidos.
  • Mercurial: sistema de control de versiones multiplataforma, para desarrolladores de software. Está implementado principalmente haciendo uso del lenguaje de programación Python.

Repositorios GIT

  • GitHub: es un sistema de gestión de proyectos y control de versiones de código, así como una plataforma de red social diseñada para desarrolladores.
  • Bitbucket: servicio de alojamiento basado en web, para los proyectos que utilizan el sistema de control de versiones Mercurial y GIT.
  • Gitlab: servicio web de control de versiones y desarrollo de software colaborativo basado en Git. Además de gestor de repositorios, el servicio ofrece también alojamiento de wikis y un sistema de seguimiento de errores, todo ello publicado bajo una Licencia de código abierto.

DESPLIEGUE

Microservicios y Message Brokers

Una “arquitectura de microservicios” es un enfoque para desarrollar una aplicación software como una serie de pequeños servicios, cada uno ejecutándose de forma autónoma y comunicándose entre sí, por ejemplo, a través de peticiones HTTP a sus API.

Un Message Broker es un programa intermediario que traduce los mensajes de un sistema desde un lenguaje a otro, a través de un medio de telecomunicaciones.

  • RabbitMQ: software de negociación de mensajes de código abierto que funciona como un middleware de mensajería.
  • Apache Kafka: aplicación multiplataforma de código abierto especializada en el procesamiento de flujos de datos.
  • Redis: sistema de almacenamiento en memoria volátil en RAM basado en registros clave-valor NoSQL. Actualmente Redis es usado por: Twitter, Instagram, GitHub, Flickr, o Digg.
  • Amazon SQS: servicio de cola de mensajes.

Monitorización

  • New Relic: sistema de motorización de máquinas que permite tener un control en tiempo real de los recursos disponibles.
  • Datadog: servicio de monitorización para aplicaciones en la nube, que proporciona monitorización de servidores, bases de datos, herramientas y servicios, a través de una plataforma de análisis de datos basada en "Software como Servicio".
  • Dynatrace: proporciona supervisión automatizada, completa y alimentada por inteligencia artificial, que brinda una visión profunda (respuestas, no datos) de cada usuario, cada transacción, en cada aplicación.

Infraestructura

  • IaaS (Infraestructura como Servicio): Es una infraestructura informática inmediata que se aprovisiona y administra a través de Internet
  • CaaS (Contenedor como Servicio): Se trata de una forma de virtualización basada en contenedores en la que los motores de contenedores, la orquestación y los recursos informáticos subyacentes se entregan a los usuarios como un servicio de un proveedor en la Nube.
  • PaaS (Plataforma como Servicio): Proporciona una plataforma y un entorno que permiten a los desarrolladores crear aplicaciones y servicios.
  • FaaS (Funciones como Servicio): Se conoce como serverless architecture. Esta arquitectura permite la ejecución de aplicaciones a través de contenedores efímeros.
  • SaaS (Software como Servicio): También conocido como software bajo demanda, o software out-of-the-box. Se trata de un servicio que ofrece a los usuarios una aplicación determinada que pueden utilizar inmediatamente, sin necesidad de instalar nada, de hacer ningún despliegue, ni de llevar su mantenimiento.

Contenedores

  • Docker: Plataforma de software que le permite crear, probar e implementar aplicaciones rápidamente. Docker empaqueta software en unidades estandarizadas llamadas contenedores que incluyen todo lo necesario para que el software se ejecute, incluidas bibliotecas, herramientas de sistema, código y tiempo de ejecución.
  • Docker compose: Permite definir las aplicaciones multicontenedor, conocidas como stacks, y ejecutarlas en un nodo individual o en un clúster.
  • Docker Swarm: Permite agrupar una serie de hosts de Docker en un clúster y gestionar los clústeres de forma centralizada así como orquestar los contenedores. Se basa en una arquitectura maestro-esclavo: mientras que el maestro de Swarm es responsable de la gestión del clúster y la delegación de tareas, el esclavo se encarga de ejecutar las unidades de trabajo (tasks o tareas).
  • Kubernetes: plataforma portable y extensible de código abierto para administrar cargas de trabajo y servicios.
