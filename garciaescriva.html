HTTP y redes

Internet
Internet es una red de Ordenadores que se encuentran interconectados a nivel mundial para compartir información.
El origen de internet se debe gracias a un proyecto militar desarrollado por ARPANET(una red del ministerio de defensa de los Estados Unidos).
Todos los ordenadores conectados a Internet deben utilizar los mismos protocolos y normas para comunicarse entre ellos(por ejemplo TCP/IP).
Internet funciona de tal manera que si alguno de sus nodos falla, puede seguir funcionando
Protocolo
Concepto: Un protocolo de comunicaciones es un sistema de reglas que permiten que dos o más entidades se comuniquen entre ellas para transmitir información. Se trata de las reglas o el estándar que define la sintaxis, semántica y sincronización de la comunicación, así como también de los métodos de recuperación de errores.
Un protocolo es un conjunto de normas que permite la comunicación entre ordenadores, estableciendo la forma de identificación de estos en la red, la forma de transmitir los datos y la forma en que la información se debe procesar.

Protocolo HTTP
Concepto, utilidad, características: Es un protocolo el cual nos permite realizar una petición de datos y recursos, como pueden ser documentos HTML. Es la base del intercambio de datos en la Web, y un protocolo de cliente-servidor. El cliente envía una petición (request) y el servidor envia una respuesta (response). El protocolo HTTP es sencillo, extensible y un protocolo con sesiones pero sin estados.
Estructura de los paquetes petición y respuesta:
Petición
Un método HTTP(por ejemplo GET o POST), La dirección del recurso pedido, La versión del protocolo HTTP, Cabeceras HTTP opcionales, Cuerpo del mensaje(en caso de algún metodo como POST o PUT).
Respuesta
La versión HTTP, Código de estado, Mensaje de estado, Cabeceras HTTP, recurso (si se ha solicitado. GET)
Métodos HTTP y su utilidad: Los principales metodos del protocolo HTTP son: GET(solicita un recurso específico, solo recibe datos), HEAD(idéntico a GET pero sin el cuerpo de la respuesta), POST(se utiliza para enviar una entidad a un recurso en específico, causando un cambio en el servidor), PUT(reemplaza el recurso por el de la petición), DELETE(borra un recurso) y PATCH(modificaciones parciales a un recurso).
Link
Proceso de Carga de una página web
  1. El navegador va al servidor DNS y encuentra la dirección IP del sitio web
  2. El navegador envía la petición HTTP al servidor
  3. Si el servidor aprueba la petición, esté enviará un mensaje(200 OK) y empezará a enviar los archivos de la pagina web al cliente en paquetes
  4. El navegador Cliente reúne los paquetes y genera el sitio web

TCP
Concepto: Es uno de los protocolos fundamentales de Internet. Los ordenadores utilizan TCP para crear conexiones entre sí a traves de las cuales puede enviarse un flujo de datos, de forma segura y en el mismo orden que se enviaron. Tambien proporciona un mecanismo para distinguir varias aplicaciones en la misma máquina a traves de los Puertos. Da soporte a protocolos como HTTP, SMTP, SSH y FTP.
Puertos Bien Conocidos: Son asignados por IANA, entre el 0 y el 1023. Suelen ser usados por el sistema o por procesos con privilegios. Las aplicaciones que usan este tipo de puertos son ejecutadas como servidores y se quedan a la escucha. Algunos ejemplos son: FTP(21), SSH(22), Telnet(23), HTTP(80)
Estructura del segmento TCP: Está compuesto por los datos enviados de la Capa de aplicación y la cabecera del protocolo de transporte. El segmento TCP es encapsulado en un datagrama IP para ser enviado a la Capa de Red.
Socket: Un socket es un proceso o hilo existente en la máquina cliente y en la máquina servdidor, que sirve para que el programa servidor y cliente lean y escriban información. Esta información se transmite por las distintas capas de Red.
Link
IP
Concepto: Protocolo de internet. Es un protocolo de comunicación de datos digitales clasificado en la Capa de Red (nivel 3 OSI). El diseño del protocolo IP se realizó presuponiendo que la entrega de los paquetes no sería confiable. Por ello, mediante técnicas de enrutamiento, buscará la mejor ruta entre las conocidas por la máquina que use IP. Los datos se envían mediante Paquetes. El IP actual es el IPv4 y su sucesor se cree que será IPv6 ya que ofrecerá mayor cantidad de direcciones.
Dirección IP: Un número que identifica de manera lógica a una interfaz de un dispositivo dentro de una red que use el protocolo IP, que corresponde al nivel de red.
Enrutamiento: El mecanismo por el que en una red los paquetes se hacen llegar desde su origen hasta su final, siguiendo una rutat a través de la red. En una red grande o en un conjunto de redes, el camino a seguir hasta llegar al destino puede suponer transitar por muchos nodos intermedios. Asociada a enrutamiento existe la métrica, con la cual se elige la mejor ruta dependiendo del uso.
Estructura de Paquete IP: Versión | IHL | DSCP | ECN | Longitud total | Identificacion | Banderas | Desplazamiento | Tiempo de vida | Protocolo | Checksum | Origen | Destino | Opciones
URL
Concepto: ( Localizador de Recursos Uniforme) Es el mecanismo usado por los navegadores para obtener cualquier recurso publicado en la web. Es una dirección que es dada a un recurso único en la web. En teoria cada recurso tiene una URL personalizada.
Estructura: Protocolo (http) | Dominio (www.example.com) | Puerto (por defecto se omite (:80) | Ruta del recurso (/path/to/file.html) | Parámetros
Parámetros: Se incluyen en la URL. Su estructura es ?variable1=valor1&variable2=valor2. Se pueden incluir tantos parametros como se necesiten. El servidor utiliza estos parámetros para realizar cosas adicionales antes de devolver el recurso.
Link
Aplicaciones cliente-servidor vs P2P
Cliente-servidor: Necesita un cliente y un servidor. El proceso depende de un servidor. Se conecta a un servidor. Se centraliza en un host
P2P: Los equipos toman ambos roles (cliente y servidor). El proceso corre sobre toda la red de ordenadores. Se conecta a una red. Suelen ser redes descentralizadas con muchos nodos.
Cookies
¿Qué son y para qué sirven? Las cookies son archivos de texto que se almacenan en el directorio del navegador. Estas se generan para hacer un seguimiento de tus movimientos por el sitio, indicarte dónde dejaste la navegación, recordar tu inicio de sesión, selección de tema, otras preferencias y personalización. A menudo son indispensables.
¿Cómo podemos establecer una sesión usando cookies? Se almacena su usuario y contraseña como una cookie y se recuerda a lo largo de diferentes visitas.
¿Dónde viaja la cookie dentro del paquete HTTP? Dentro de las cabeceras HTTP de respuesta y de petición.
Link
Tipos MIME
Es un forma estandarizada de indicar la naturaleza y formato de un documento, archivo o conjunto de datos. Los navegadores usan el tipo MIME para determinar como se procesará un documento, así que es importante que los servidores estén bien configurados para devolver el MIME en el encabezado de respuesta.

Proyectos Web

Requerimientos, especificaciones de una aplicación
La especificación de requisitos de Software es una descripción completa del comportamiento del sistema que se va a desarrollar. Incluye requisitos funcionales y no funcionales. Funcionales, aquellos servicios que el sistema debe proporcionar y No Funcionales, que definen propiedades o restricciones de los servicios o funciones ofrecidos por el sistema.
Metodologías del desarrollo de software
Es un marco de trabajo usado para estructurar, planificar y controlar el proceso de desarrollo en sistemas de información.
  • Modelo en cascada: Metodología lineal
  • Modelo en espiral: Combinación de metodología lineal e iterativa
  • Modelos basados en prototipos: Metodología iterativa
  • Metodologías agiles: Metodologías basadas en el desarrollo iterativo e incremental, donde los requisitos y soluciones evolucionan con el tiempo segun la necesidad del proyecto.
    • Métodos kanban: Método para gestionar el trabajo intelectual, con énfasis en la entrega justo a tiempo, mientras no se sobrecarguen los miembros del equipo.
    • Extreme programming: Método que pone más énfasis en la adaptabilidad que en la previsibilidad. Los cambios sobre la marcha son un aspecto inevitable e incluso deseable
    • Metodología Scrum: Marco de trabajo para desarrollo de software.
      • Sprints: Es el período en el cual el trabajo se lleva a cabo, al final de cada sprint el equipo debe mostrar los avances logrados y lo que se obtiene es un producto potencialmente entregable para el cliente. Suele durar entre 2 y 4 semanas. Fases: Planificación del Sprint, Scrum diario, Revisión de Sprint y Retrospectiva del Sprint
      • Diagrama burnout: Es una tabla que mide la cantidad de requisitos del proyecto pendientes al comienzo de cada Sprint. Lo normal es que la gráfica vaya descendiendo hasta que el proyecto se termine.
      • Herramientas Scrum: Las herramientas son los elementos que garantizan la transparencia y el registro de la información del proceso Scrum. Ayudan a la productividad y la calidad de los proyectos. PRODUCT BACKLOG: Una lista de los requerimientos del producto. SPRINT BACKLOG: Subconjunto de requerimientos del Product Backlog que se eligen para abordarse en un Sprint. INCREMENTO: Se trata del resultado del Sprint, es decir la entrega Potencialmente Desplegable.

Integración continua
Una práctica de Ingeniería de software que consiste en hacer integraciones automáticas de un proeycto lo más a menudo posible para poder detectar fallos cuanto antes. La integración es la compilación y ejecución de pruebas de un proyecto. SERVIDORES: Jenkins, Bamboo, CircleCI, GitLab, Apache Continuum, CruiseControl.
Pruebas Unitarias: Consisten en aislar una parte del código y comprobar que funciona correctamente. Herramientas: xUnit, Junit, NUnit, PHPUnit. Mock: objeto que imita el comportamiento de objetos reales de una forma controlada.
Pruebas de integración: Las pruebas que se realizan una vez se aprueban las pruebas unitarias que comprueban si las partes del código funcionan juntos correctamente
TDD, Desarrollo dirigido por tests: Una práctica que involucra Escribir las pruebas primero y Refactorización. Primero se desarrolla el test unitario, y luego se implementa el código que hace que funcione. Se usa para obtener un código funcional y limpio.

Conjunto de Documentos que incluyen con un Software, que contienen las instrucciones para que el usuario conozca sus funciones principales. Puede ser en Papel, en soporte informático o incluso Multimedia.
Tipos: Documentación de Ayuda(cliente) Documentación de Soporte(para resolver los errores más comunes) Documentación técnica(especificaciones y requisitos) Documentación General, Documentación Legal,etc
Herramientas: Javadoc, PHPdoc, KDoc, Ndoc
Despliegue en producción y en test. Despliegue automatizado. Despliegue en la nube
En producción: El software se pone en funcionamiento real para el cliente. En test: El software se despliega para que los equipos de pruebas lo verifiquen. Automatizado: El Software se mueve entre Producción y Tests de manera automática. En la nube: Se refiere a desplegar la aplicación como un Servicio para el cliente.

El lugar en el que se almacenan los datos actualizados e históricos de cambio, a menudo en un servidor

Evaluar las características funcionalidad y confiabilidad del producto, según los objetivos fijados, para detectar las debilidades del mismo para alimentar un proceso de mejora continua.

Normalización
Nuestra Base de datos debe cumplir los requisitos naturales para funcionar optimamente sin perjudicar al rendimiento. Estos son:
  • Primera Forma Normal: No repetir datos en nuestras tablas
  • Segunda Forma Normal: Cada columna debe depender de la clave, si hay columnas que se repiten para la misma clave, es mejor generar otra tabla
  • Tercera Forma Normal: Cada tabla debe tener columnas relacionadas a su clave y no otras.
Restricciones de Integridad
La Clave primaria. Claves externas. Valores no nulos. Unicos
BD VS SGBD - SGBDR
Una Base de datos es un conjunto de datos, almacenados en un soporte informático, organizados de forma independiente de su utilización y accesibles por distintos usuarios y aplicaciones. Un Sistema Gestor de Bases de Datos es un conjunto de programas que permite definir, manipular y utilizar la información que contienen las bases de datos, se podría considerar al SGBD como la interfaz entre el usuario y la BD
SGBDR: Sistema Gestor de Bases de Datos Relacionales, Un tipo de base de datos en el que los datos se almacenan en tablas.
DML-DDL:SQL
DDL: Lenguaje de Definición de Datos, Comandos SQL que se usan para definir el esquema de la base de datos. Por ejemplo CREATE, DROP, TRUNCATE.
DML: Lenguaje de Manipulación de Datos, Comandos SQL que se usan para modificar los datos presentes en la base de datos. Por ejemplo INSERT, UPDATE, DELETE.
noSQL
También llamado No solo SQL, son SGBD que no usan SQL como el lenguaje principal de consultas. Los datos no requieren tablas. La ventaja que tiene contra SQL es que son más fácilmente escalables.

Wireframers: Un esquema visual que representa el esqueleto o una estructura visual de un sitio web.

Procesamiento asíncrono, Callback, Callback Hell, Promesas y Programación reactiva
No se esperan a las instrucciones diferidas y se continúa ejecutando el código

  • Callback Hell: Al tratar de escribir Javascript de una forma en el que la ejecución ocurra de forma visible de arriba a abajo, se acaba generando un churro de código
  • Promesas: Es un proxy para un valor que no necesariamente se conoce al momento de ser Creada. Permite asociar callbacks que se ejecutarán en función al exito o fracaso de la promesa.
  • Programación Reactiva: Paradigma enfocado en el trabajo con flujos de datos finitos o infinitos de manera asíncrona, permitiendo que estos datos se propaguen generando cambios en la aplicación, es decir, "reaccionan" a los datos ejecutando una serie de eventos

Programación y organización del código

Concepto de código spaguetti. Explicar porqué se necesita una estructuración del código
Codigo spaguetti. Código que básicamente no sabes dónde empieza ni donde acaba. Se necesita una mejor estructuración para mejorar la claridad, calidad y tiempo de desarrollo.
Concepto de Abstracción en programación
Paradigmas de la programación
Un paradigma de programación es un estilo de desarrollo de programas. Es decir, un modelo para resolver problemas computacionales.
  • Paradigma Declarativo frente al paradigma imperativo: En el imperativo los programas se componen de un conjunto de sentencias que cambian de estado. Son secuencias de comandos que ordenan acciones a la máquina. El declarativo, todo lo contrario, se basa en describir el problema que se quiere solucionar.
  • ¿Qué es la programación funcional y en qué se diferencia respecto de la programación imperativa?: La diferencia entre la programación funcional y la programación imperativa es que la programación funcional considera los cálculos como funciones matemáticas y evita el cambio de estado y los datos mutables, mientras que la programación imperativa usa las declaraciones que cambian el estado de los programas.
Técnicas de programación imperativa
  • Programación estructurada: Paradigma de programación orientado a mejorar la claridad, calidad y tiempo de desarrollo de un programa de computadora recurriendo únicamente a subrutinas y tres estructuras básicas: secuencia, selección (if y switch) e iteración (bucles for y while).
  • Programación Modular: Paradigma de programación que consiste en dividir un programa en módulos o subprogramas para hacerlo más legible y manejable. Acoplamiento: El grado de dependencia de los modulos. Cohesión: Los modulos relacionados van juntos
  • Programación Orientada a Objetos: Paradigma de programación en que los Objetos se utilizan como metáfora para emular las entidades reales.
    • Principios SOLID: Guías que pueden ser aplicadas en el desarrollo de Software para eliminar malos diseños provocando que el programador tenga que refactorizar el código hasta que sea legible y extensible. Principio de Responsabilidad Única, Principio de abierto/cerrado, Principio de sustitucion de Liskov, Principio de Segregación de la interfaz, Principio de la inversión de la Dependencia
    • Interfaces: Una interfaz es un conjunto de métodos abstractos y de constantes cuya funcionalidad es la de determinar el funcionamiento de una clase, es decir, funciona como un molde o como una plantilla. Al ser sus métodos abstractos estos no tiene funcionalidad alguna, sólo se definen su tipo, argumento y tipo de retorno.
    • Clases: Representan un tipo particular de Objetos. Atributos y Métodos
      • Clases Abstractas: Clases de las que no puedes generar una Instancia. Clases que pueden servir como padre de otras o incluso Clases que implementan métodos abstractos.
      • Propiedades y Métodos: Visibilidad de los métodos y de las propiedades: Propiedades: Son los atributos de la Clase (variables). Métodos: Implementan funcionalidad asociada al objeto, equivalen a las funciones en programación estructurada. En las Clases el acceso a variables no debe permitirse directamente, si no a traves de sus métodos get y set.

Regla de la Genericidad del Código
Cuanto más genéricos son los programas, más reusable es el código pero también es más complejo y difícil de mantener.
Cuanto menos generico sean los programas, menos reusable es el código pero es más sencillo y fácil de mantener.
Lo que se busca es un termino medio entre ambos. Que sean programas medianamente genéricos con código medianamente reusable con dificultad/facilidad media de programación y mantenimiento medianamente fácil.

Lenguajes de programación

Lenguajes de programación
  • Lenguajes compilados vs interpretados : En los lenguajes Compilados, el código fuente debe pasar por un compilador antes de ser ejecutado. En los lenguajes interpretados, el intérprete lo que hace es leer el archivo y ejecutar linea por linea lo que entiende. En un lenguaje compilado, cada vez que hagamos un cambio debemos volver a compilar el programa. Los programas compilados para cierto sistema operativo lo más probable es que no funcionen en otro, en cambio los lenguajes interpretados son normalmente multiplataforma. El compilado es más rapido en ejecutarse, ya que en el interpretado debe volverse a interpretar el código
  • Compilador, Interprete y Transpilador: El Compilador genera un programa stand-alone, El interprete interpreta cada vez el código, El transpilador es como el Compilador pero en vez de traducirse a un lenguaje de un menor nivel de abstracción, se traduce a uno del mismo nivel(el caso de typescript a javascript)
Lenguajes débilmente tipados vs fuertemente tipados: diferencias, ventajas y desventajas
En los lenguajes debilmente tipados no indicamos el tipo de variable al declararla, e incluso podemos asignar un valor entero a una variable que tenia una cadena. VENTAJAS: No se declara el tipo, Podemos cambiar el tipo sobre la marcha, Se escribe menos código. DESVENTAJAS: Al hacer operaciones a veces salen mal, en JavaScript al comparar números a veces se comparan como cadena, Casteos, Inseguridad
En los lenguajes fuertemente tipados se indica el tipo de variable al declararla. El tipo no puede ser cambiado y no podremos operar entre distintos tipos. VENTAJAS: Código más expresivo y menos errores por tipos de variable. DESVENTAJAS: Se escribe más código.
Diferencia entre tipado estático y dinámico
La principal diferencia entre los dos es que en el tipado estático nosotros definimos el tipo de la variable desde el principio y en el tipado Dinámico el tipo es definido al vuelo por el mismo lenguaje (nos da facilidad de no definir los tipos al declarar una variable)
Timetomarket de un lenguaje
Es el tiempo comprendido desde que un producto es concebido hasta que está disponible para el usuario final
¿En qué Consiste el type typing?
En typeScript debemos describir el formato de los objetos y de los datos en cuanto a tipos de variable
¿Qué es la sobrecarga de operadores?
Consiste en modificar los operadores como por ejemplo + y - , para poder usarlos en las Clases que hemos definido. Se llama sobrecarga ya que utilizaremos el mismo operador para un número de usos diferentes, y el compilador, dependiendo sobre qué se opera, elige como usarlo.
¿Qué es la interpolación de cadenas?
Es el proceso de evaluar una Cadena literal que contiene marcadores de posición, lo que produce un resultado en el que los marcadores de posición se reemplazan por sus valores correspondientes.

  • Java Virtual Machine
    • ¿Qué es? Es la máquina virtual de Java. Es una maquina virtual ejecutable en una plataforma especifica, capaz de interpretar y ejecutar instrucciones expresadas en un código binario especial, el cual es generado por el compilador del lenguaje Java.
    • Ventajas y Desventajas: Java se ejecuta en una zona controlada, el uso de la JVM nos permite utilizar bytecodes en cualquier dispositivo con JVM. Los programas escritos en Java pueden ser más lentos que los escritos en C++ por ejemplo.
